# Render Deployment Troubleshooting Guide

## Understanding the "No open ports detected" Error

This error occurs when your FastAPI application fails to start or crashes before binding to the PORT that Render provides.

## Common Causes

### 1. **Missing Environment Variables** ❌
The most common cause. The app crashes during startup because required services (Supabase, OpenRouter) aren't configured.

**Solution:**
- Verify all environment variables are set in Render dashboard:
  - `DATABASE_URL`
  - `SUPABASE_URL`
  - `SUPABASE_SERVICE_KEY`
  - `OPENROUTER_API_KEY`

### 2. **Import/Dependency Issues** ❌
A required Python package might not be installed or there's a syntax error.

**Solution:**
```bash
# Test locally
python scripts/validate_startup.py

# Check for errors
pip install -r requirements.txt
python -c "from app.main import app; print('✓ OK')"
```

### 3. **Database Connection Issues** ❌
The app can't connect to Supabase/PostgreSQL at startup.

**Solution:**
- Test the DATABASE_URL locally
- Verify Supabase credentials
- Check if database is initialized

### 4. **Port Binding Issue** ❌
The app isn't listening on all interfaces (should be 0.0.0.0, not localhost).

**Solution:** Already configured in render.yaml:
```yaml
startCommand: uvicorn app.main:app --host 0.0.0.0 --port $PORT
```

## How to Debug

### Step 1: Local Validation
```bash
# Activate virtual environment
.venv\Scripts\Activate.ps1

# Run validation script
python scripts/validate_startup.py

# Try importing app directly
python -c "from app.main import app; print(app)"
```

### Step 2: Test Locally with Environment Simulation
```bash
# Create test .env file
cp .env .env.test

# Run uvicorn with explicit port
uvicorn app.main:app --host 0.0.0.0 --port 8000

# Test endpoints in another terminal
curl http://localhost:8000/
curl http://localhost:8000/health
curl http://localhost:8000/health/detailed
```

### Step 3: Check Render Logs
In Render Dashboard:
1. Go to your service
2. Click "Logs" tab
3. Look for:
   - Build errors
   - Runtime errors
   - Module import failures
   - Environment variable warnings

### Step 4: Validate render.yaml
```bash
# Basic syntax check
cat render.yaml | python -c "import sys; import yaml; yaml.safe_load(sys.stdin); print('✓ Valid')"
```

## Health Check Endpoints

After deployment, these endpoints help verify your service is running:

- **`GET /`** - Basic status check
- **`GET /health`** - Simple health check
- **`GET /health/detailed`** - Full diagnostics with environment info

Example:
```bash
curl https://your-service.onrender.com/health/detailed
```

Expected response:
```json
{
  "status": "ok",
  "service": "chronicle-ai-backend",
  "environment": {
    "database_configured": true,
    "supabase_configured": true,
    "openrouter_configured": true,
    "cors_origins": "configured"
  }
}
```

## Pre-Deployment Checklist

- [ ] All required environment variables set
- [ ] `requirements.txt` up to date
- [ ] Run `python scripts/validate_startup.py` locally - all checks pass
- [ ] Tested locally: `uvicorn app.main:app --host 0.0.0.0 --port 8000`
- [ ] All routes accessible at `http://localhost:8000/docs`
- [ ] Database can be connected to
- [ ] API keys (OpenRouter) are valid

## Environment Variables Reference

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `SUPABASE_URL` | Yes | Supabase project URL |
| `SUPABASE_SERVICE_KEY` | Yes | Supabase service key |
| `OPENROUTER_API_KEY` | Yes | OpenRouter API key |
| `SUPABASE_BUCKET` | No | Storage bucket name |
| `OPENROUTER_MODEL` | No | LLM model (default: google/gemma-4-31b-it:free) |
| `CORS_ORIGINS` | No | Allowed origins (default: http://localhost:5173) |
| `LOG_LEVEL` | No | Logging level (default: info) |

## Quick Fix Steps

**If you see "Timed Out" error:**

1. ✅ Run local validation: `python scripts/validate_startup.py`
2. ✅ Fix any reported issues
3. ✅ Test locally with: `uvicorn app.main:app --host 0.0.0.0 --port 8000`
4. ✅ Ensure `GET /health` returns 200 OK
5. ✅ Push changes to git
6. ✅ Trigger re-deploy in Render dashboard
7. ✅ Check `/health/detailed` endpoint after deploy
